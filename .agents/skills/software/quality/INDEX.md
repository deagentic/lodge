# Software Quality Skills

## Forward-TDD Pipeline (use in this order for new features)
- [tech-lead](tech-lead/SKILL.md) — story intake gate, task decomposition, merge approval
- [bdd-writer](bdd-writer/SKILL.md) — Gherkin scenarios BEFORE any code is written
- [tdd-developer](tdd-developer/SKILL.md) — Red→Green→Refactor enforcer
- [code-reviewer](code-reviewer/SKILL.md) — quality, security, and TDD discipline check
- [qa-validator](qa-validator/SKILL.md) — scenario coverage audit + exploratory testing, last gate before merge
